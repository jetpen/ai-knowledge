# Introduction to Ontologies

Ontologies are structured systems for knowledge representation, acting as "smart organizers" that define concepts, categories, properties, and relationships within a data domain. They enable both computers and people to understand, connect, and retrieve information effectively.

## Core Components
An ontology is constructed using four main building blocks:
1.  **Individuals**: Specific instances (e.g., specific movies like "Titanic").
2.  **Categories (Classes)**: General groupings (e.g., "Actors," "Directors").
3.  **Properties**: Attributes of an item (e.g., runtime, budget).
4.  **Relationships**: Connections between individuals or classes (e.g., "Leonardo DiCaprio *starred-in* Titanic").

## Why They Are Necessary
Ontologies provide the semantic context computers need to process ambiguous queries, transform raw data into a structured knowledge graph, and facilitate improved search engine results, voice assistant comprehension, and recommendation systems.

## Common Ontology Languages
*   **Web Ontology Language (OWL)**: Standard for web-based systems.
*   **Open Biomedical Ontologies (OBO)**: Domain-specific for medicine/biology.
*   **Rule Interchange Format (RIF)**: Facilitates system interoperability.
*   **CycL**: Historical system designed for complex logical relationships.

## Applications
*   **Search**: Precise entity resolution.
*   **AI Assistants**: Understanding voice intent.
*   **Recommendation Systems**: Context-aware content suggestions.
*   **Healthcare**: Computer-aided diagnosis and clinical logic.
