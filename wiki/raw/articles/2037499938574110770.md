# Agentic Memory: A Detailed Breakdown

## Overview
Ramakrishna defines Agentic Memory as a system that enables LLMs to evolve from stateless chatbots into agents capable of remembering, learning, and building over time.

## Three Roles of Memory
1. **Continuity**: Identity and user preferences.
2. **Context**: Task-specific state (what just happened, tool outputs, next steps).
3. **Learning**: Improving decisions over time (avoiding repetition).

## The Four Types of Memory
### 1. In-context Memory
*   **Definition**: The context window; the "working desk."
*   **Contents**: System prompts, history, tool results, scratchpad, retrieved memories.
*   **The Window Problem**: History eventually overflows.
*   **Mitigation Strategies**: Summarization (compressing old turns), selective retention (discarding small talk), and offloading to external storage.

### 2. External Memory
*   **Definition**: Persistent storage outside the model.
*   **Structured Store (Exact Lookup)**: PostgreSQL, Redis, etc. Best for user profiles and structured data.
*   **Vector Stores (Semantic Search)**: Pinecone, Chroma, etc. Best for unstructured notes and episodic recall.
*   **Key Insight**: Retrieval is the bottleneck; good architecture is 80% retrieval design.
