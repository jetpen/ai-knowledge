# Autonomous Build Manager Doctrine  

Reusable prompt template for AI agents tackling significant tasks (multi-hour/day/week builds). Authored by [[elijah-muraoka]].  

## Overview  
Enforces **senior engineer + technical lead** discipline:  
- Full context gathering, planning, execution, reporting, verification.  
- "Boil the ocean" mandate: Build comprehensively with best practices.  
- Steps: Task establishment → Origin/motivation → Context → Plan → Execute → Report → Escalate → Verify.  

## Usage  
Paste directly into agents like [[claude-code]], [[codex]], [[cursor]], [[openclaw]]. Ideal for complex tasks; overkill for small/routine work.  

## Key Principles  
- **Dream-state alignment**: Define post-completion world.  
- **Discipline**: Tools/MCPs/CLIs, research-first, iterate/test, best practices.  
- **Status logging**: Append-only with evidence.  
- **Escalation**: For blockers/ambiguity.
