# Stewardship Pattern

Organizational contract for compounding long-horizon AI work: each session updates and improves shared structure, so individual “harnesses” reinforce (rather than fragment) into a better org-level system.

## Definition (from the X article)
Everyone can keep their own harness (Claude/Codex/etc.). The contract is that every session leaves the shared structure in better shape than it started by updating structure, adding missing pieces, and introspecting artifacts.

## Core behaviors
- Update shared structure (not only final outputs)
- Add missing artifacts where they’re absent in the org system
- Introspect how decisions/documents were produced
- Prefer governance of the shared system over fragmented private learning

## Relation to compounding and decay
- Enables measurable compounding.
- Prevents “AI decay” / fragmentation when org learning isn’t explicit and governed.
