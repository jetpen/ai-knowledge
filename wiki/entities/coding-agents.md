# Coding Agents

## Overview
Coding agents are specialized AI systems designed to automate software engineering tasks. They leverage large language models (LLMs) to understand, write, modify, debug, and test code across complex software projects. Unlike simple code completion tools, coding agents are typically autonomous or semi-autonomous, possessing broader agency to execute shell commands, navigate file systems, and integrate with development environments.

## Key Facts
- **Core Functionality**: Autonomously perform tasks like bug fixing, feature implementation, refactoring, and project-wide documentation.
- **Capabilities**: Integration with IDEs (like VS Code), file system access, terminal execution, and context-awareness of the entire codebase via RAG (Retrieval-Augmented Generation) or large context windows.
- **Tool Use**: Frequently utilize tools for linting, testing, Git operations, and web-based research to facilitate development.

## Relationships
- **Powered by**: Foundation models (like Claude 3.5 Sonnet, GPT-4o).
- **Embedded in**: Development workflows, often complementing human developers.
- **Related Technologies**: [Browserbase](/wiki/entities/browserbase.md) (for web-related research/testing), [E2B](/wiki/entities/e2b.md) (for secure sandboxed execution).
