# x402 — HTTP-Native Payments Protocol

## Overview
`x402` is an open, neutral internet-native payment standard that leverages the `HTTP 402 Payment Required` status code. It enables frictionless, machine-to-machine programmatic value transfer without accounts, KYC, or credentials. It functions as the settlement/transport layer of the agentic commerce stack.

## Core Mechanism
- **HTTP native**: Built directly into existing HTTP request/response cycle.
- **402 semantics**: Server returns `402 Payment Required` with payment details when access is gated.
- **Client retry**: Client issues payment (typically stablecoin on supported networks) and retries original request.
- **Minimal middleware**: One line of integration (`paymentMiddleware(...)`) gates any route.

## Key Advantages
- Zero protocol fees beyond network gas.
- Instant settlement at internet speed.
- No onboarding friction.
- Fully open and network-agnostic.

## Ecosystem Context
- Designed primarily for **agentic commerce** — autonomous AI agents paying for services.
- Complementary to mandate/intent layers such as [[agent-payments-protocol]].
- Supported by major infrastructure and fintech players: AWS, Vercel, Cloudflare, Stripe.

## Relationships
- Provides payment execution and settlement for [[agent-payments-protocol]] (human-not-present samples use `x402` directly).
- Layers beneath higher-level agent coordination protocols ([[a2a]], [[mcp]]).
- Payments primitive enabling [[agentic-infrastructure]].
