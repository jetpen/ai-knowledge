# SSE (Server-Sent Events)

SSE is a standard that allows servers to push real-time updates to web clients over a single, long-lived HTTP connection. In the context of agentic AI protocols, SSE is a critical transport mechanism for streaming interactions between agents, clients, and services.

## Role in Agentic Ecosystems
SSE serves as a key asynchronous communication primitive:
- **Low Latency:** Enables real-time streaming of text, data, and status updates as they are generated by an agent.
- **Protocol Integration:** Extends HTTP/1.1 and HTTP/2 for bidirectional-like flow, where the server holds the connection open to push events (e.g., token-by-token generation in LLM responses).
- **A2A Protocol Interaction:** The Agent2Agent (A2A) protocol leverages SSE as a primary mechanism for flexible interaction, specifically supporting streaming responses where multi-turn agent collaboration requires persistent, low-overhead communication channels.
- **Efficiency:** More lightweight than WebSockets for use cases that primarily require server-to-client updates (one-way streaming), making it ideal for standardizing agent-to-user or agent-to-agent interfaces without the overhead of full-duplex socket management.

## Key Concepts
- **Long-lived Connection:** A single HTTP request with `Content-Type: text/event-stream` that remains open.
- **Event Stream:** A series of text-based event chunks (Data, ID, Event type, Retry interval).
- **Capability:** Commonly used in FastAPI (`fastapi.sse`), Python MCP (`mcp.client.sse`, `mcp.server.sse`), and many other modern agent frameworks to handle stream-based updates.
