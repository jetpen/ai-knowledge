# AG-UI (Agent-User Interaction Protocol)

**AG-UI** is an open, lightweight, event-based protocol designed to standardize the connection between AI agents and user-facing applications. It acts as a middleware layer to facilitate real-time context, agent execution events, and user interface updates.

## Overview
AG-UI bridges the gap between agent backends and frontend applications, ensuring interoperability across diverse frameworks and transport mechanisms.

### Positioning in the Protocol Stack
- **MCP**: Provides agents with tools.
- **A2A**: Enables agent-to-agent communication.
- **AG-UI**: Bridges agents directly into user-facing applications.

## Key Features
- **Event-Based**: Supports standardized event formats for agent-user synchronization.
- **Transport Agnostic**: Compatibility with SSE, WebSockets, and webhooks.
- **Generative UI**: Native support for building dynamic, structured messaging interfaces.
- **Bi-directional Sync**: Real-time state synchronization and context enrichment.
- **Human-in-the-Loop**: Native support for collaborative workflows between agents and humans.

## Ecosystem Partners
- **Frameworks**: LangGraph, CrewAI, Microsoft Agent Framework, Google ADK, LlamaIndex, Agno, Pydantic AI.
- **Language Support**: Broad support via community SDKs (Kotlin, Golang, Dart, Java, Rust, Ruby, C++).
