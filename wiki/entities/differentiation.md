# Differentiation

**Agentic Coding Pattern** from [[agentic-coding-patterns]] (AiPatternBook REPL).

## Context
## Agentic Application

**Differentiation**, in \"Product Judgment and What to Create\", helps agents identify unique value by contrasting with competitors, using competitive landscape analysis.

**Forces**:
- Commodity AI solutions erode margins.
- Fast follower risks in open-source agents.
- Need evidence-based moats (tech, data, UX).

**Solution**:
1. Cross-reference landscape survey with value prop.
2. LLM extracts gaps: e.g., LangChain lacks built-in idempotency at scale.
3. Propose differentiators: proprietary modules, hybrid orchestration.
4. Validate via prototypes, user sims.
5. Agents tool-use: GitHub issues mining for pain points.

**Examples**:
- **Framework Diff**: Vs. CrewAI, emphasize graph cycles for loops.
- **Enterprise Agent**: Differentiate with compliance (SOC2) modules.
- **Personal AI**: UX-first voice+vision over text-only rivals.
- **Benchmark Diff**: Run evals showing 2x speed on task-decomp.

Creates sustainable edges; surveys show differentiated agents retain 3x users.
