# Compute and Sandboxing

## Overview
Compute and sandboxing refer to the critical infrastructure required to run untrusted or dynamic code generated by AI agents safely and efficiently. This domain includes serverless runtimes, containerized environments, and isolation layers that prevent agent-executed code from accessing sensitive host system resources.

## Key Facts
- **Purpose**: Enables AI agents to execute arbitrary code (e.g., Python, shell scripts) as part of their reasoning or task-completion process.
- **Security**: Focuses on isolation, resource limitation (CPU/memory), and preventing malicious or accidental system compromise.
- **Popular Approaches**:
    - Micro-VMs (like Firecracker)
    - Container isolation
    - WASM runtimes
